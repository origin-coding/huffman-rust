# Huffman 算法
Huffman 编码是一种广泛用于无损数据压缩的算法。它基于字符出现的频率来构建最优的前缀编码，从而让常见字符使用较短的编码，而不常见的字符使用较长的编码，从而达到压缩数据的目的。
## 算法原理
Huffman 算法的核心是构建一颗 Huffman 树，这是一种特殊的二叉树，每个叶子节点代表一个字符及其在文本中出现的频率。树的构建过程如下：
1. **初始化**：将每个字符视为一个节点，并按照出现的频率排序，形成一个新的优先队列。
2. **构建树**：每次从队列中取出两个频率最低的节点，创建一个新的节点作为父节点，父节点的频率是两个子节点频率之和，然后将新节点重新加入队列。
3. **重复步骤**：重复上述过程，直到队列中只剩下一个节点，这个节点就是 Huffman 树的新节点。
## 为什么这样做是最优解
Huffman 算法是一种贪心算法，它能保证频率高的字符距离根节点更近（编码更短）。我们使用**带权路径长度（WPL）** 来衡量这颗树的优劣（即压缩效率）：
$$WPL=\sum_{k=1}^{n}{w_k}\times{l_k}$$
其中：
- $n$ 是字符种类的数量
- $w_k$ 是第 $k$ 个字符的频率
- $l_k$ 是第 $k$ 个字符的编码长度（路径长度）
Huffman 树保证了在所有可能的二叉树构建方式中，计算出的 **WPL 值是最小的**，这意味着编码后的总二进制流也是最短的，从而实现了最高的压缩效率。
## 编码过程
一旦 Huffman 树构建完成，就可以通过从根节点到每个叶子节点的路径来确定每个字符的 Huffman 编码。路径中左转路径表示为“0”，右转表示为“1”，这样每个字符都会被分配到一个唯一的二进制编码，及 Huffman 编码。
## 解码过程
解码时，从 Huffman 树的根节点开始，根据编码表中的每一位进行遍历，知道到达叶节点，即可找到响应的字符。由于 Huffman 编码是前缀码，因此这个过程不会有歧义。
# 项目结构搭建
在开始写算法之前，我们先搭建项目的基本结构。
为了让项目具有良好的扩展性，方便后续实现 GLI 和 GUI，我们不应该把所有的代码都放在 `main.rs`，我们将采用 Rust 的 **Workspace（工作空间）** 模式。
我们的目标目录结构如下：
```text
huffman-rust/           # 项目根目录
├── Cargo.toml          # Workspace 配置文件
└── crates/             # 存放所有后端代码
    └── huffman_core/   # 核心库 (算法、文件结构定义)
```
## 初始化工作空间
首先，创建工作目录。注意，这里我们**不使用** `cargo new` ，因为根目录只是一个容器，并不包含源代码。
```bash
mkdir huffman-rust
cd huffman-rust
git init
```
接下来，在根目录下创建一个 `Cargo.toml` 文件。这个文件告诉 Rust：这是一个工作空间，包含以下成员。
```toml
[workspace]
members = [
	"crates/huffman_core", 
	# "crates/huffman_cli", <-- 未来我们会添加这个 
	# "crates/huffman_gui", <-- 还有这个
]

resolver = "3"
```
## 创建核心算法库
我们将 Huffman 算法逻辑放在一个独立的 Library 中，这样 CLI 和 GUI 都可以使用它。
我们将所有的代码都放在 `crates/` 目录下，这是一种常见的 Rust 代码组织习惯。
```bash
mkdir crates
# 注意使用 --lib，因为我们要创建一个库，而不是可执行程序
cargo new --lib huffman_core
```
现在项目的文件结构应该已经成型了。
## 引入必要的依赖
为了不重复造轮子，我们需要引入几个需要用到的 crate。进入 `crates/huffman_core` 目录，修改其中的 `Cargo.toml`：
```toml
[package]  
name = "huffman_core"  
version = "0.1.0"  
edition = "2024"  
  
[dependencies]  
binrw = "0.15.0"  
thiserror = "2.0.18"  
anyhow = "1.0.101"
```
## 验证环境
回到项目根目录，运行构建命令来确保一切配置正确：
```bash
cd ../..
cargo build
```
如果你看到命令行输出 `Finished `dev` profile [unoptimized + debuginfo] target(s) in 2.58` 一样的内容，就表明项目已经创建完成，我们可以继续后续的开发工作。
